# PCSE/WOFOST 作物模型使用指南

## 一、这是什么

PCSE（Python Crop Simulation Environment）是荷兰瓦赫宁根大学开发的作物模拟框架，核心模型是 WOFOST（WOrld FOod STudies）。它是一个**生物物理过程模拟器**——逐日模拟作物从出苗到成熟的生长发育过程，而非简单的产量计算器。

### 模型能做什么

| 能力 | 说明 |
|------|------|
| 物候模拟 | 逐日推算发育阶段（DVS），预测出苗、开花、成熟的具体日期 |
| 生物量累积 | 基于光截获→光合作用→呼吸消耗→干物质分配的完整链条 |
| 产量差距分析 | 对比潜在产量（光温决定上限）和水限产量，拆分水分亏缺造成的减产 |
| 水平衡全链路追踪 | 降雨→入渗→蒸发→蒸腾→渗漏→径流，每滴水都记账 |
| 假设实验 | 换个播种日期、换个品种、换个灌溉策略，2 秒跑完一整季 |
| 极端天气评估 | 用 30 年历史天气各跑一遍，量化干旱/冷害风险 |
| 品种筛选 | 同一地点换不同品种参数，看哪个最优 |
| 灌溉决策 | 土壤水分动态实时跟踪，知道什么时候该浇、浇多少 |

### 模型不能做什么

- **不是天气预报**：逐日天气是输入，不是输出
- **不是病虫害模型**：没有病害/虫害模块
- **不是温室模型**：不模拟温度分层、VPD、CO₂ 施肥、补光响应等温室特有过程

## 三、可用模型

`pcse/models.py` 中定义了 18 个预配置模型：

```
Wofost72_PP              # WOFOST 7.2  潜在生产（光温决定）
Wofost72_WLP_CWB         # WOFOST 7.2  水限生产 + 经典水平衡 ← 最常用
Wofost72_Phenology       # WOFOST 7.2  仅物候
Wofost73_PP/WLP_CWB/WLP_MLWB   # WOFOST 7.3 三个版本
Wofost81_PP/WLP_CWB/WLP_MLWB/NWLP_CWB_CNB/NWLP_MLWB_CNB/NWLP_MLWB_SNOMIN  # 7个版本
Lintul10_NWLP_CWB_CNB    # LINTUL3 氮水限制
Lingra10_PP/WLP_CWB/NWLP_CWB_CNB  # LINGRA 草地模型 3 个版本
FAO_WRSI10_WLP_CWB       # FAO 水分满足指数
Alcepas10_PP             # 芦笋模型
```

命名规则：`PP`=潜在生产 `WLP`=水限 `NWLP`=氮水双限 `CWB`=经典水平衡 `MLWB`=多层水平衡 `CNB`=经典氮平衡

## 四、环境要求与常见问题

### Python 版本兼容性

PCSE 6.0.13 原生面向 Python 3.9-3.10。当前环境是 Python 3.13.5 —— 已修复以下兼容性问题：

| 问题 | 原因 | 修复 |
|------|------|------|
| `unittest.makeSuite` 不存在 | Python 3.13 删除了该 API | 替换为 `unittest.TestLoader().loadTestsFromTestCase()` |
| pandas `Series[0]` 报 KeyError | pandas 3.x 严格了索引类型检查 | `.loc[ix, var][0]` 改为 `.loc[ix, var].iloc[0]` |

修复涉及的文件：`pcse/tests/test_*.py`（9 个文件）+ `tests/run_tests.py`

### 其他已知警告

- `RequestsDependencyWarning: urllib3 (2.6.3) or chardet (7.4.3) doesn't match` — requests 库版本兼容问题，不影响模拟结果
- `MatplotlibDeprecationWarning: plot_date was deprecated` — 仅影响画图，不影响模拟

## 六、能输出哪些变量

### 逐日输出（每天一个值，12 个默认变量）

| 变量 | 含义 | 单位 |
|------|------|------|
| DVS | 发育阶段（0=出苗, 1=开花, 2=成熟） | - |
| LAI | 叶面积指数 | m²/m² |
| TAGP | 总地上生物量 | kg/ha |
| TWSO | 贮藏器官干重（籽粒/块茎） | kg/ha |
| TWLV | 叶片干重 | kg/ha |
| TWST | 茎秆干重 | kg/ha |
| TWRT | 根系干重 | kg/ha |
| TRA | 实际蒸腾速率 | cm/day |
| RD | 根深 | cm |
| SM | 根区土壤含水量 | - |
| WWLOW | 根区可用水量 | cm |
| RFTRA | 蒸腾衰减因子（水分胁迫指标，1=无胁迫） | - |

| 变量 | 含义 |
|------|------|
| LAIMAX | 全季最大叶面积指数 |
| CTRAT | 冠层温度比率 |
| DOS | 播种日期 |
| DOE | 出苗日期 |
| DOA | 开花日期 |
| DOM | 成熟日期 |
| DOH | 收获日期 |
| DOV | 品种特定日期 |
| CEVST | 累计蒸散量 |

### 终端输出（模拟结束时输出一次，9 个变量）

| 变量 | 含义 |
|------|------|
| WTRAT | 总水分胁迫 |
| EVST | 土壤蒸发总计 |
| EVWT | 作物蒸腾总计 |

## 八、模拟的核心流程

WOFOST 逐日循环的物理过程：

```
每日天气输入（Tmax, Tmin, Rain, Rad, Wind, Vap）
    │
    ├─→ 物候模块：积温→DVS推进→判断出苗/开花/成熟
    │
    ├─→ 叶动态模块：DVS驱动LAI变化（出叶→衰老→凋落）
    │
    ├─→ 光截获：LAI→截获辐射→潜在光合速率
    │
    ├─→ 同化模块：光合速率×光截获→总同化量（考虑温度/水分胁迫）
    │
    ├─→ 呼吸模块：维持呼吸消耗（各器官×温度系数）
    │
    ├─→ 分配模块：DVS驱动→净同化物分配到根/茎/叶/籽粒
    │
    ├─→ 蒸散模块：Penman-Monteith→潜在蒸散→实际蒸腾（受土壤水分限制）
    │
    └─→ 水平衡模块：降雨-蒸发-蒸腾-渗漏→更新土壤含水量
```

每个模块都是可替换的——修改配置文件中的组件名即可切换不同的光合模型、蒸散公式或水平衡方案。

## 十、跑自己的数据需要什么

用 WOFOST 模拟自己的地块，需要准备 4 类输入：

### 1. 天气数据（WeatherDataProvider）

| 方式 | 适用场景 |
|------|---------|
| `NASAPowerWeatherDataProvider(lat, lon)` | 任意地点，NASA 在线拉取 |
| `ExcelWeatherDataProvider(file.xlsx)` | 自己的实测数据，Excel 格式 |
| `CSVWeatherDataProvider(file.csv)` | 自己的实测数据，CSV 格式 |
| `CABOWeatherDataProvider(file)` | 传统 CABO 格式（WOFOST Control Centre） |
| `GridWeatherDataProvider(db, grid_no)` | CGMS 网格数据（从数据库读） |

### 2. 作物参数（CropData）

- CABO 格式：`CABOFileReader('cropfile.crop')`
- PCSE 格式：`PCSEFileReader('cropfile.crop')`
- 直接字典：`{'TSUM1': 1100, 'TSUM2': 900, ...}`

### 3. 土壤参数（SoilData）

- CABO 格式：`CABOFileReader('soilfile.soil')`
- PCSE 格式：`PCSEFileReader('soilfile.soil')`

### 4. 种植管理（AgroManagement）

YAML 格式，定义出苗/收获日期和任何定时/状态触发事件：

```yaml
- 2000-01-01:                    # 模拟开始日期
    CropCalendar:
        crop_name: wheat
        variety_name: winter-wheat
        crop_start_date: 2000-01-01
        crop_start_type: emergence
        crop_end_date: 2000-10-01
        crop_end_type: harvest
        max_duration: 300
    TimedEvents: null
    StateEvents: null
```

## 十一、常用代码片段

### 创建自定义模拟

```python
from pcse.input import NASAPowerWeatherDataProvider, CABOFileReader, YAMLAgroManagementReader
from pcse.input import WOFOST72SiteDataProvider
from pcse.base import ParameterProvider
from pcse.models import Wofost72_WLP_CWB

# 天气（NASA 在线）
wdp = NASAPowerWeatherDataProvider(latitude=52, longitude=5)

# 参数
cropdata = CABOFileReader('mycrop.crop')
soildata = CABOFileReader('mysoil.soil')
sitedata = WOFOST72SiteDataProvider(WAV=100)
params = ParameterProvider(cropdata=cropdata, soildata=soildata, sitedata=sitedata)

# 种植管理
agro = YAMLAgroManagementReader('mycalendar.agro')

# 模拟
model = Wofost72_WLP_CWB(params, wdp, agro)
model.run_till_terminate()
```

### 输出到 DataFrame

```python
import pandas as pd
output = model.get_output()
df = pd.DataFrame(output).set_index('day')
print(df.describe())
df.to_excel('results.xlsx')
```

### 切换作物品种

```python
# crop=1 冬小麦, crop=2 玉米, crop=3 春大麦
w = pcse.start_wofost(crop=2, mode='pp')  # 玉米潜在生产
w.run_till_terminate()
```
