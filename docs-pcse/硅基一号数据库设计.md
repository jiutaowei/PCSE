## 一、基础字典表

### 1.1 device — 种植设备

对应 PCSE: `grid` 表。一台设备 = 一个种植箱。

| 字段 | 类型 | 说明 |
|------|------|------|
| `device_id` | INTEGER PK | 自增主键 |
| `device_code` | VARCHAR(20) UNIQUE | 设备编号，如 `S1-001` |
| `device_name` | VARCHAR(100) | 设备名称，如 "1号种植箱" |
| `location` | VARCHAR(100) | 物理位置 |
| `plant_date` | DATE | 首次投入使用日期 |
| `status` | VARCHAR(20) | `active` / `maintenance` / `offline` |

### 1.2 crop_type — 作物类型

对应 PCSE: `crop` 表。作物大类，如番茄、草莓、生菜。

| 字段 | 类型 | 说明 |
|------|------|------|
| `crop_id` | INTEGER PK | 自增主键 |
| `crop_name` | VARCHAR(50) | 番茄 / 草莓 / 生菜 |
| `crop_code` | VARCHAR(20) UNIQUE | `tomato` / `strawberry` / `lettuce` |
| `growth_type` | VARCHAR(20) | `indeterminate`（无限生长）/ `determinate`（有限） |

### 1.3 variety — 品种

对应 PCSE: `variety`。同一作物下的不同品种，参数继承覆盖作物默认值。

| 字段 | 类型 | 说明 |
|------|------|------|
| `variety_id` | INTEGER PK | 自增主键 |
| `crop_id` | INTEGER FK → crop_type | 所属作物 |
| `variety_name` | VARCHAR(100) | 如 "樱桃番茄 F1" |
| `variety_code` | VARCHAR(50) | 如 `cherry_f1` |
| `is_default` | INTEGER | 是否为该作物的默认品种 |
| `breeding_company` | VARCHAR(100) | 育种公司 |

### 1.4 substrate_type — 基质类型

对应 PCSE: `soil_type` 表。

| 字段 | 类型 | 说明 |
|------|------|------|
| `substrate_id` | INTEGER PK | 自增主键 |
| `substrate_name` | VARCHAR(100) | 椰糠 / 岩棉 / 泥炭混合 |
| `substrate_code` | VARCHAR(50) | `coco_coir` / `rockwool` / `peat_mix` |

## 二、参数表 (Key-Value 模式)

> **这是 PCSE 最核心的设计思想：参数做成行而不是列。** 加新参数不需要 `ALTER TABLE`，品种间不会出现大片 NULL 列。

### 2.1 crop_params — 作物参数

对应 PCSE: `crop_parameter_value`。

| 字段 | 类型 | 说明 |
|------|------|------|
| `crop_id` | INTEGER FK → crop_type | 所属作物 |
| `param_code` | VARCHAR(50) | 参数名，如 `TSUM1` `TBASE` `SLA` `LUE` |
| `param_value` | REAL | 参数值 |
| `dvs_from` | REAL | 发育阶段起始，默认 0 |
| `dvs_to` | REAL | 发育阶段结束，默认 2 |
| `unit` | VARCHAR(20) | 单位 |

```sql
-- 示例：番茄的积温参数
INSERT INTO crop_params (crop_id, param_code, param_value, unit)
VALUES (1, 'TSUM1', 850, '°C·d'),       -- 营养生长所需积温
       (1, 'TSUM2', 1100, '°C·d'),      -- 生殖生长所需积温
       (1, 'TBASE', 10, '°C'),           -- 基础温度
       (1, 'SLA', 0.025, 'm²/g');        -- 比叶面积
```

### 2.2 variety_params — 品种参数（覆盖作物默认值）

对应 PCSE: `variety_parameter_value`。

| 字段 | 类型 | 说明 |
|------|------|------|
| `variety_id` | INTEGER FK → variety | 所属品种 |
| `param_code` | VARCHAR(50) | 参数名 |
| `param_value` | REAL | 覆盖值 |

**继承覆盖逻辑:**

```
查 crop_params    → 获得作物级默认值
查 variety_params → 如果品种定义了同名参数，覆盖之
查 device_config  → 如果设备有特殊配置，再覆盖（如定植密度）
```

### 2.3 substrate_params — 基质物理参数

对应 PCSE: `soil_physical_group`。

| 字段 | 类型 | 说明 |
|------|------|------|
| `substrate_id` | INTEGER FK → substrate_type | 基质类型 |
| `param_code` | VARCHAR(50) | `WCST`（饱和含水量）/ `WCFC`（田间持水量）/ `WCWP`（萎蔫点）/ `KSAT`（饱和导水率） |
| `param_value` | REAL | 参数值 |

### 2.4 device_config — 设备配置

对应 PCSE: `site` 表。每台设备的运行参数，如目标 VPD、EC 范围、补光阈值等。

| 字段 | 类型 | 说明 |
|------|------|------|
| `device_id` | INTEGER FK → device | 设备 ID |
| `param_code` | VARCHAR(50) | 配置项名 |
| `param_value` | TEXT | 配置值（字符串以兼容非数值型） |

## 三、种植周期

### 3.1 growth_cycle — 种植茬次

对应 PCSE: `crop_calendar`。**重大改造：从一次性 → 多轮采收。**

| 字段 | 类型 | 说明 |
|------|------|------|
| `cycle_id` | INTEGER PK | 茬次 ID |
| `device_id` | INTEGER FK → device | 所在设备 |
| `variety_id` | INTEGER FK → variety | 种植品种 |
| `cycle_name` | VARCHAR(100) | 如 "2026-春茬 樱桃番茄" |
| `sow_date` | DATE | 播种日期 |
| `transplant_date` | DATE | 定植日期 |
| `first_flower_date` | DATE | 始花日期 |
| `first_harvest_date` | DATE | 首次采收 |
| `end_date` | DATE | 拉秧日期（NULL=进行中） |
| `max_truss` | INTEGER | 计划留果穗数 |
| `harvest_rounds` | INTEGER | 已完成采收轮次 |
| `truss_removed` | INTEGER | 已打顶穗数 |
| `status` | VARCHAR(20) | `planned` / `active` / `completed` / `failed` |

**与 PCSE 的关键区别:** 番茄是无限生长型，不存在 DVS=2 就死掉的情况。`harvest_rounds` 持续累加，直到 `end_date` 或 `max_truss` 触发拉秧。

## 四、传感器数据

### 4.1 sensor_readings — 传感器原始读数

对应 PCSE: `grid_weather`。**重大改造：天气变量 → 温室多传感器。** 双主键 `(device_id, recorded_at)` 保证跨设备时序查询效率。

| 分类 | 字段 | 类型 | 说明 |
|------|------|------|------|
| 空气环境 | `air_temp` | REAL | 空气温度 (°C) |
| | `air_humidity` | REAL | 相对湿度 (%) |
| | `vpd` | REAL | 饱和水汽压差 (kPa) |
| | `co2_ppm` | INTEGER | CO₂ 浓度 (ppm) |
| 光照 | `par_umol` | REAL | 光合有效辐射 (μmol/m²/s) |
| | `solar_rad` | REAL | 太阳总辐射 (W/m²) |
| | `led_par` | REAL | LED 补光 PAR |
| | `light_hours` | REAL | 当日光照时长 (h) |
| 基质/根区 | `substrate_temp` | REAL | 基质温度 (°C) |
| | `substrate_moist` | REAL | 基质含水量 (%) |
| | `ec_ms` | REAL | 电导率 (mS/cm) |
| | `ph` | REAL | pH 值 |
| 营养液 | `nutrient_temp` | REAL | 营养液温度 (°C) |
| | `nutrient_ec` | REAL | 营养液 EC (mS/cm) |
| | `nutrient_ph` | REAL | 营养液 pH |
| 系统状态 | `water_level` | REAL | 水箱液位 (%) |
| | `pump_status` | INTEGER | 水泵 (0/1) |
| | `fan_status` | INTEGER | 风扇 |
| | `light_status` | INTEGER | 补光灯 |
| | `heater_status` | INTEGER | 加热器 |

索引: `CREATE INDEX idx_sensor_device_ts ON sensor_readings(device_id, recorded_at);`

### 4.2 sensor_hourly — 小时聚合（可选）

| 字段 | 类型 | 说明 |
|------|------|------|
| `device_id` | INTEGER FK | 设备 ID |
| `hour_start` | TIMESTAMP | 小时起始时间 |
| `air_temp_avg/min/max` | REAL | 温度统计 |
| `humidity_avg` | REAL | 湿度均值 |
| `vpd_avg` | REAL | VPD 均值 |
| `co2_avg` | REAL | CO₂ 均值 |
| `par_sum` | REAL | 累积 PAR（DLI 计算用） |

> 说明：如果传感器采集频率超过 1 次/小时，用此表做第一层聚合，减少原始表体积。`par_sum` 是日累积光量（DLI = Daily Light Integral）的计算基础。

## 五、模拟/模型输出

### 5.1 sim_output_timeseries — 逐日输出

对应 PCSE: `sim_results_timeseries`。

| 分类 | 字段 | 类型 | 说明 |
|------|------|------|------|
| 基础 | `cycle_id` | INTEGER FK | 茬次 |
| | `sim_day` | INTEGER | 模拟第几天（相对天数） |
| | `calendar_date` | DATE | 实际日期 |
| | `sim_mode` | VARCHAR(10) | `pp` / `wlp` / `nwlp` |
| 发育 | `DVS` | REAL | 发育阶段 |
| | `leaf_count` | INTEGER | 叶片数 |
| 生物量 | `TAGP` | REAL | 总地上生物量 |
| | `WLV / WST / WRT` | REAL | 叶/茎/根干重 |
| | `WSO` | REAL | **果实干重（番茄关键指标）** |
| | `WSO_cumulative` | REAL | 累计采收果实干重 |
| 光截获 | `LAI` | REAL | 叶面积指数 |
| | `PAR_absorbed` | REAL | 冠层吸收 PAR (MJ/m²/d) |
| | `GDD_cumulative` | REAL | 累计有效积温 (°C·d) |
| 水分 | `TRA` | REAL | 实际蒸腾 |
| | `WC` | REAL | 基质含水量 |

### 5.2 cycle_summary — 茬次摘要

对应 PCSE: `sim_results_summary`。

| 分类 | 字段 | 说明 |
|------|------|------|
| 关键日期 | `emergence_date` | 出苗 |
| | `anthesis_date` | 开花 |
| | `first_harvest` | 首次采收 |
| | `maturity_date` | 拉秧 |
| 产量 | `total_yield` | 总产量 (g/株 或 kg/m²) |
| | `fruit_count` | 总果实数 |
| | `avg_fruit_weight` | 平均单果重 (g) |
| | `harvest_rounds` | 采收轮次 |
| | `yield_per_truss` | 每穗产量 |
| 生物量 | `TAGP_max / LAI_max / DVS_final` | 峰值指标 |
| 效率 | `WUE` | 水分利用效率 (g 果实 / L 水) |
| | `LUE` | 光能利用效率 (g 生物量 / MJ PAR) |
| | `NUE` | 氮素利用效率 (g 生物量 / g N) |
| 环境 | `total_water / total_par / gdd_total / avg_vpd_day` | 全季累计与均值 |

### 5.3 gdd_tracking — GDD 积温追踪

独立于模拟输出，与硅基一号现有的 GDD 判定系统配合。

| 字段 | 说明 |
|------|------|
| `gdd_daily` | 当日有效积温 |
| `gdd_cumulative` | 累计积温 |
| `current_phase` | 当前阶段: `seedling` / `vegetative` / `flowering` / `fruiting` / `harvest` |
| `phase_progress` | 当前阶段完成百分比 (0-100) |
| `phase_gdd_required` | 当前阶段所需总积温 |
| `estimated_next_phase` | 预计进入下一阶段日期 |
| `estimated_maturity` | 预计成熟日期 |

## 六、控制与操作记录

### 6.1 control_actions — 控制动作

| 字段 | 类型 | 说明 |
|------|------|------|
| `device_id` | INTEGER FK | 设备 |
| `cycle_id` | INTEGER FK | 茬次 |
| `action_time` | TIMESTAMP | 执行时间 |

| `drain_ec` | 排出液 EC |
| `drain_ratio` | 排出率 (%) — 无土栽培关键健康指标 |

## 七、AI 决策与知识库

### 7.1 decision_log — 决策记录

完整记录 AI 推理链路 P→S→W→D→E→V（感知→状态→世界模型→决策→执行→验证）。

| 字段 | 类型 | 说明 |
|------|------|------|
| `perception_json` | TEXT | 传感器快照 |
| `state_json` | TEXT | 状态表征 |
| `world_model_json` | TEXT | 世界模型推演 |
| `decision_json` | TEXT | 冲突消解 + 决策 |
| `execution_json` | TEXT | 执行动作 |
| `verification_json` | TEXT | 验证结果 |
| `human_rating` | INTEGER | 人工评分 (1-10) |
| `ai_confidence` | REAL | AI 置信度 |
| `is_success` | INTEGER | 是否成功 |

### 7.2 knowledge_base — 经验知识库

从成功/失败决策中沉淀可复用规则。

| 字段 | 说明 |
|------|------|
| `knowledge_type` | `rule` / `pattern` / `lesson` / `insight` |
| `trigger_condition` | 触发条件 |
| `action_desc` | 推荐操作 |
| `rationale` | 原理说明 |
| `success_count / fail_count` | 成败计数 |
| `confidence` | 综合置信度（默认 0.5） |
| `status` | `active` / `deprecated` / `superseded` |

## 建库

```powershell
python -c "
import sqlite3
db = sqlite3.connect(r'D:\work\code\silicon1\pcse\doc\downloads\硅基一号.db')
with open(r'D:\work\code\silicon1\pcse\doc\downloads\硅基一号数据库设计.sql', encoding='utf-8') as f:
    db.executescript(f.read())
db.commit()
print('硅基一号数据库已创建')
"
```
